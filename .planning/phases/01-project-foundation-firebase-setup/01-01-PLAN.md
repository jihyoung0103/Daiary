<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gradle/libs.versions.toml → build scripts | Version catalog values control which binary artifacts are downloaded from Maven repositories |
| gradle/wrapper/gradle-wrapper.properties → Gradle distribution | distributionUrl controls which Gradle binary is fetched on first build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | gradle-wrapper.properties distributionUrl | mitigate | Use official services.gradle.org URL with exact version pin (9.4.1); validateDistributionUrl=true is set to reject modified distributions |
| T-01-02 | Information Disclosure | google-services.json | mitigate | Add app/google-services.json to .gitignore before first commit (Phase 1 checkpoint task handles this); Firebase API keys in this file are low-risk alone but must be protected from open Security Rules pairing |
| T-01-03 | Tampering | libs.versions.toml dependency versions | accept | Maven Central and Google Maven require HTTPS; dependency integrity is enforced by Gradle checksum verification for wrapper; no custom Maven repos used |
</threat_model>
